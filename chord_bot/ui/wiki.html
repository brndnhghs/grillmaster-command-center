01 / CHORDS & INTERVALS

Building Blocks: Notes, Intervals, and Chord Qualities

A chord is three or more notes played simultaneously. The distances between those notes — measured in semitones — determine the chord's quality. Chord Bot uses six core qualities, each with a distinct sound and harmonic role.

02 / SCALES & MODES

Modal Color

The Tonic node sets the key and mode — the ordered pattern of whole and half steps that defines which pitches are available. Each mode has a distinct emotional character. Click a mode and hit ▶ to hear the scale ascend.

SCALE GALAXY — lit nodes are the scale degrees; inner ring shows the pattern of whole (W) and half (H) steps

Dorian is minor with a raised 6th — gives it that jazzy, modal feel. Mixolydian is major with a flat 7th — sounds bluesy and open.

03 / HARMONIC FUNCTIONS

The Markov Model

Every chord belongs to a harmonic function: Tonic (home), Subdominant (departure), Dominant (tension), or Pre-dominant (approach). The Function node uses a Markov chain to pick the next function probabilistically — music that coheres without repeating.

Tonic — rest, home (T)
Subdominant — motion (S)
Dominant — tension, pull (D)
Pre-dominant — approach (P)

Dominant resolves to Tonic 60% of the time. Pre-dominant drives almost exclusively to Dominant (70%). The machine never gets stuck — it always has somewhere to go.

04 / TENSION

Shaping the Arc

Every chord carries a tension value from 0.0 (stable) to 1.0 (unstable). The Tension Shaper augmenter overrides this value independently of chord choice. Drag the handles below — the chord you hear adapts to the tension level at each slot.

TENSION ARC — drag any handle; release to hear the chord quality at that tension level

A dramatic arc: start at 0 (Tonic), rise through 0.5 (passing motion), peak at 0.9 (Dominant 7♭9), resolve back to 0.

05 / THE GRAPH MODEL

H-axis and V-axis

Chord Bot organizes music as a two-dimensional node graph. Horizontal (H-axis) nodes are connected left to right and generate the chord sequence. Vertical (V-axis) nodes hang below H-axis nodes and modify their output — they never change the sequence, only how it sounds.

The executor walks the H-chain left to right, threading a HarmonicState object through each node. At each H-node, any attached V-nodes process the state in stack order before it moves on. The result is a list of timed chord events that drive the audio engine and piano roll.
